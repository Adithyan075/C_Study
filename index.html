What is Matrix Multiplication?

📖 Definition

Matrix multiplication is a binary operation that produces a matrix from two matrices. Each element of the result is the dot product of a row from the first matrix and a column from the second matrix.

🔢 Formula

For matrices A (m×n) and B (n×p):

C[i][j] = Σ A[i][k] × B[k][j]

where k ranges from 0 to n-1

🔑 Key Rules

  • Take Row i from Matrix A (left to right)
  • Take Column j from Matrix B (top to bottom)
  • Multiply corresponding elements pairwise

Section 3 · Matrix in C

Common Errors to Avoid

❌ Not Initializing C to 0
int C[N][N]; // garbage values!
int C[N][N] = {0}; // ✅ always initialize

Without initialization, C may contain random memory values, giving wrong results.

❌ Wrong Loop Order
C[j][i] += A[i][k] * B[k][j]; // wrong!
C[i][j] += A[i][k] * B[k][j]; // ✅ correct

Mixing up indices causes computing wrong elements of C.

❌ Forgetting += (using = instead)
C[i][j] = A[i][k] * B[k][j]; // overrides!
C[i][j] += A[i][k] * B[k][j]; // ✅ accumulate

Using = instead of += loses the accumulated sum — only keeps the last product.

❌ Incompatible Matrix Sizes
// Multiplying A(2×3) as if it were A(3×3)
// Always check: cols of A == rows of B ✅

Always verify dimensions before coding the multiplication.

📖 Definition

Sorting is the process of arranging elements in a specific order — either ascending (smallest to largest) or descending (largest to smallest).

📈
Ascending: [1, 3, 5, 7, 9] — smallest first
📉
Descending: [9, 7, 5, 3, 1] — largest first
🔍
Why sort? Binary search only works on sorted data. Databases sort for fast queries.

Section 4 · Array Sorting

Bubble Sort — How It Works

The Core Idea 💡

Compare adjacent elements. If they're in the wrong order, swap them. Repeat this for every pair, from left to right. After one full pass, the largest element "bubbles" to the end.

Pass-by-Pass Example: [5, 3, 8, 1, 2]

Pass 1:
[5,3,8,1,2]→[3,5,8,1,2]→[3,5,8,1,2]→[3,5,1,8,2]→[3,5,1,2,8]
Pass 2:
[3,5,1,2,8]→[3,5,1,2,8]→[3,1,5,2,8]→[3,1,2,5,8]
Pass 3:
[1,3,2,5,8]→[1,3,2,5,8]→[1,2,3,5,8]
Pass 4:
[1,2,3,5,8] ✅ Already sorted!

Section 5 · Sorting in C

Optimized Bubble Sort

🚀 Early Exit Optimization

Add a swapped flag. If no swaps occur in a pass, the array is already sorted — exit early!

Best case improves: O(n²) → O(n)
✅
Set swapped = 0 at the start of each pass
✅
Set swapped = 1 whenever a swap occurs
✅
After each pass, check: if swapped == 0, break
optimized_bubble.c 
for(i=0; i<n-1; i++) {
  int swapped = 0;
  for(j=0; j<n-i-1; j++) {
    if(arr[j] > arr[j+1]) {
      temp = arr[j];
      arr[j] = arr[j+1];
      arr[j+1] = temp;
      swapped = 1;
    }
  }
  if(swapped == 0) break; // ✅ Early exit
}

Key Concepts Summary

🔢

Matrix Multiplication

  • A(m×n) × B(n×p) = C(m×p)
  • C[i][j] = sum of A[i][k] × B[k][j]
  • Cols of A must equal Rows of B
  • O(n³) time complexity
💻

Matrix in C

  • Declare: int C[N][N] = {0}
  • Triple nested loop (i, j, k)
  • Use += to accumulate
  • Print with nested printf loop
🔀

Bubble Sort

  • Compare adjacent elements
  • Swap if in wrong order
  • n-1 passes for n elements
  • O(n²) worst, O(n) optimized best
⚡

Sorting in C

  • Ascending: if(arr[j] > arr[j+1])
  • Descending: if(arr[j] < arr[j+1])
  • 3-variable swap with temp
  • Optimize with swapped flag
